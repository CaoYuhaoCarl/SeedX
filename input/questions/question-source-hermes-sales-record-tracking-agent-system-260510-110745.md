如何用 Hermes Agent 做追踪销售记录的 agent 系统？

希望产出一个可落地的多 Agent 系统设计，用于记录、追踪、整理和复盘销售过程中的客户线索、跟进记录、成交进展、阻塞点、下一步行动和复购/转介绍机会。

关注点：
- 如何拆分 Agent 角色与工作流；
- 如何利用 Hermes Agent 的 gateway、file tools、memory、skills、cron、delegation，以及可能的表格/CRM/飞书/Notion 等连接；
- 如何设计输入输出文件结构和销售记录字段；
- 如何保证记录准确、去重、提醒及时、复盘有效；
- 先给 MVP，再给逐步升级路线。
