# 学习问题

我想快速掌握 AI Agent Harness Engineering 的基本框架，并判断它如何帮助长期运行的 Agent 工作流更稳定、可观察、可修复。

## 背景

我已经知道可以让 LLM 调用工具和子任务，但不清楚当任务很长、步骤很多、需要多轮修正时，为什么需要 harness，以及 harness 应该包含哪些组件。

## 目标

1. 理解 Agent Harness Engineering 的基本定义和问题域。
2. 掌握输入协议、状态文件、日志、子 Agent 分工、评估报告、resume 修正循环等核心机制。
3. 能判断一个长任务是否需要 harness，而不是简单 prompt 或脚本。
4. 能设计一个最小可行的 file-backed agent harness。

## 时间约束

希望在 2 小时内建立可用框架，不追求具体框架 API 对比或生产级部署。

## 输出偏好

希望输出包含：

- 核心概念地图
- 分阶段学习路径
- 具体练习
- 掌握检查
- 通用应用与迁移计划
