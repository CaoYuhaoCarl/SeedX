# 学习问题

如何用 OpenClaw / Hermes Agent 搭建一个整理读书笔记、做个人 Wiki 的 Agent 系统？

## 期望输出

请从系统设计、Agent 分工、输入输出协议、知识结构、去重链接、质量评估、MVP 到扩展路线，生成一套可执行的学习与搭建路径。

## 场景约束

- 面向想把书籍、摘录、读书心得、划线内容、语音/图片笔记整理成个人 Wiki 的学习者。
- 希望系统能把原始笔记转成结构化 Markdown / Obsidian 风格条目，并自动建立概念链接、人物/书籍/主题索引。
- 希望比较 OpenClaw 与 Hermes Agent 在此类工作流中的适用方式：CLI、工具调用、长期记忆、skills、cron/定时、文件系统、消息入口等。
- 从低成本本地 MVP 开始，再逐步扩展到多书籍、多格式输入、定期复盘、知识图谱和分享输出。
- 产物应尽量具体，避免只停留在概念层。
