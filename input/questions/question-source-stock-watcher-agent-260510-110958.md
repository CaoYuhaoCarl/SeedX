# 学习问题

如何用 Agent 搭建一个盯股票的 Agent 系统？

## 期望输出

请从系统设计、Agent 分工、数据源、触发机制、风险控制、质量评估、MVP 到扩展路线，生成一套可执行的学习与搭建路径。

## 场景约束

- 面向想用 Agent 自动追踪股票、自选股、财报、公告、新闻、价格/成交量异常和市场情绪的学习者。
- 系统目标不是自动荐股或代替投资决策，而是帮助监控信息、整理信号、生成提醒、形成研究笔记。
- 希望区分行情数据、基本面数据、新闻舆情、公司公告、研报/财报等不同数据源。
- 希望设计可从低成本 MVP 开始，再扩展到多市场、多股票池、多策略、多级告警和研究知识库。
- 必须强调合规、风险提示、不可承诺收益、避免把 LLM 输出当成交易建议。
- 产物应尽量具体，避免只停留在概念层。
